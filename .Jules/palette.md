## 2026-04-26 - Add ARIA Labels and Focus Styles to Icon-only Chatbot Buttons
**Learning:** Found that `<motion.button>` and native `<button>` elements that only contain `<span className="material-symbols-outlined">...</span>` are invisible to screen readers without an `aria-label`, and further, screen readers will incorrectly read out the text ligature (e.g., "close", "chat", "send") as content unless the span is marked with `aria-hidden="true"`. Finally, Framer Motion buttons need proper `focus-visible` Tailwind classes applied directly to their `className` to ensure keyboard navigability.
**Action:** Always add `aria-label` to icon-only interactive elements, apply `aria-hidden="true"` to the inner icon spans, and add `focus-visible:ring-2` focus states for keyboard accessibility, especially for `motion.button` components.
