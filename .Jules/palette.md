## 2026-04-21 - [Framer Motion & Tailwind Focus Accessibility]
**Learning:** Icon-only buttons implemented with Framer Motion (`<motion.button>`) frequently lack proper semantic labeling and keyboard focus indicators, making them completely inaccessible. The Material Symbols ligatures (e.g., "close") inside these buttons are also read aloud by screen readers if not hidden.
**Action:** Always add `aria-label` to icon-only buttons. Add `aria-hidden="true"` to `<span className="material-symbols-outlined">` elements within buttons. Crucially, Framer Motion elements can easily support keyboard accessibility without custom CSS by directly passing standard Tailwind utility classes like `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-[color]` to their `className`.
