## 2024-04-24 - Accessibility for Icon-only Buttons (Chatbot)
**Learning:** Found a specific pattern in this app's components (`Chatbot.tsx`) where icon-only buttons built with Material Symbols lack essential accessibility context. The raw text of the material symbol (e.g., "close", "chat", "send") acts as the icon but is read as raw text by screen readers unless hidden with `aria-hidden="true"`. Also, these interactive elements rely purely on `hover` effects, missing crucial `focus-visible` states which makes keyboard navigation difficult for users relying on tabbing.
**Action:** When creating or fixing icon-only buttons (especially using Material Symbols ligatures), always apply `aria-hidden="true"` to the inner icon span, add a descriptive `aria-label` to the parent button/motion.button, and ensure clear `focus-visible` styling (e.g., `focus-visible:ring-2 focus-visible:outline-none`) to support robust keyboard accessibility.
