## 2024-06-25 - Prevent UI Blocking on Frequent Timer Ticks
**Learning:** Components with interval-based state updates (like the 100ms timer in `Dashboard` inside `src/app/page.tsx`) cause rapid, high-frequency render cycles. During these ticks, any heavy derived data calculations (e.g., mapping, reducing large object arrays like `extractionFeed`, or parsing configuration objects like `promptData`) will block the main thread synchronously, resulting in a noticeably lagging UI and unresponsive experience for the user.
**Action:** When a component features frequent state updates or ticks, strictly apply `useMemo` to decouple heavy derived data calculations from the regular render cycle, and stabilize reference props passed to children using `useCallback`. This ensures the rapid state update only triggers layout repaints for the progress bar/timer rather than full recalculations of component context.
