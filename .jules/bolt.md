## 2024-04-13 - Dashboard 100ms Timer Performance Optimization
**Learning:** The `Dashboard` component in `src/app/page.tsx` utilizes a 100ms interval timer (`setInterval` updating `extractionTimeMs`) during active extraction. This extremely frequent state update triggers full component re-renders 10 times per second. Derived state computations (like parsing prompt mappings or mapping/reducing arrays for `currentFetchedAnswers`) inside the component body are completely unmemoized, resulting in synchronous UI blocking and lag during these ticks.
**Action:** Always strictly memoize derived data (`useMemo`) and inline functions passed as props (`useCallback`) in components that feature frequent state ticks (like progress bars or timers) to decouple heavy computations from simple visual updates.
