## 2025-04-16 - Prevent Information Disclosure in API Error Responses
**Vulnerability:** API endpoints (e.g., extract, chat, refine, convert-to-markdown, generate, upload) were returning internal error details (`msg` containing `error.message` or `String(error)`) to the client via `NextResponse.json({ error: "...", details: msg })`. This could expose sensitive internal configurations, database structures, or stack traces to external users.
**Learning:** Returning detailed error messages directly to the client violates the "Fail securely" principle and the memory rule: "To prevent information disclosure, API routes must fail securely by returning generic error messages to the client... and logging detailed error messages... only server-side via `auditLog` or `console.error`."
**Prevention:** Only generic error messages (e.g., `{ error: "Operation failed" }`) should be returned to clients. Detailed error contexts must exclusively be logged server-side using tools like `auditLog` or `console.error`.
